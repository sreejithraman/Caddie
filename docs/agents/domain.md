# Domain docs

Caddie is a single-context repository.

Before exploring or changing the system, read the Caddie skill's self-contained domain reference at `skills/caddie/references/domain.md` and any relevant decisions under `docs/adr/`. If an expected document does not exist, proceed without treating its absence as a problem.

Use the glossary's canonical terms in issues, tests, specifications, and implementation. Avoid synonyms that the glossary explicitly rejects. If work conflicts with an existing ADR, surface the conflict instead of silently overriding the decision.
